2Claudeを知ろう3分で読む

KEY CONCEPT
Claudeとは
Anthropic社が開発したAIアシスタントです。テキストでの会話だけでなく、ファイルの作成、Web検索、コードの実行など、さまざまな「作業」をこなせます。ChatGPTやGeminiと同じカテゴリですが、長文の正確な処理と指示への忠実さに特に強みがあります。

3つのモデル

Claudeには用途に応じた3つのモデルがあります。車で言えば、軽自動車・セダン・高級車のような関係です。

Haiku
速い・軽い・安い
シンプルな質問、定型処理、大量の繰り返し作業に。応答が速く、コストが低い。
Sonnet
バランス型・日常使い
ほとんどの業務に最適。速度・品質・コストのバランスが良く、普段使いの定番。
Opus
最高性能・深い思考
複雑な分析、長文の構造化、高度な判断が必要なタスクに。じっくり考えて高品質な回答を出す。
ワークショップでは: 普段はSonnetが使われることが多いですが、今日のCowork作業ではOpusまたはSonnetが自動的に選ばれます。モデルの違いを意識しなくても大丈夫です。

Claudeの4つの使い方

Claudeには使い方（インターフェース）が4種類あります。同じAIですが、「何を任せたいか」によって最適な窓口が異なります。

チャット（claude.ai）
誰でもすぐ使える
ブラウザで質問・相談・文章作成。一問一答の「会話」が中心。ファイルのアップロードもできるが、手元に直接返すのは苦手。
Cowork
非エンジニア向けの作業モード
デスクトップアプリから使う。フォルダを指定して「調べて・読んで・ファイルを作って」を丸ごと任せる。今日のメイン。
Claude Code
エンジニア向け
ターミナル（黒い画面）から使う。コードを書く・テストする・デプロイするなど、開発者の相棒。
API
開発チーム向け
自社のアプリやシステムにClaudeを組み込む。裏側でClaudeが動く仕組みを作るための接続口。

3今日使うCoworkの特徴

全体像が分かったところで、今日のメインであるCoworkをもう少し詳しく見ておきましょう。

ファイルを直接読む・作る
フォルダを指定すると、中のファイルを読み込み、Excel・Word・PDFなどを作って返してくれます。
Webで調べてまとめる
「この3社を比較して」と頼むだけで、Web検索して情報を集め、表やレポートにまとめてくれます。
長時間の自律作業
50ファイルの処理のような時間のかかる作業も、放っておけば最後までやり切ります。
サイドバーで進捗が見える
今何をやっているかがリアルタイムで分かり、途中で止めることもできます。

ハンズオン

5課題①（7分）ハンバーガーチェーン3社の比較

日本のハンバーガーチェーン3社（マクドナルド、モスバーガー、バーガーキング）について、Webで最新の公開情報を調べてもらい、①価格帯、②ターゲット層、③強み・特徴、④店舗展開、の4軸で比較表（Excel）を作ってください。あわせて、3社の違いがひと目で分かる1ページのサマリー（Word）も作ってください。

この課題のポイント: 素材ファイルはありません。Coworkが自分でWebを検索し、情報を集め、ファイルにまとめるところまで全部やってくれます。さきほどの表で言う「Web検索」と「ファイル操作」を両方使う体験です。

期待される成果物:

  • 比較表（.xlsx）
  • 1ページサマリー（.docx）

プロンプト例（そのままコピーして使えます）:

日本のハンバーガーチェーン3社（マクドナルド、モスバーガー、バーガーキング）について、 Webで最新情報を調べて、以下2つのファイルを作ってください。 1. comparison.xlsx — 比較表 列: 企業名 / 価格帯 / ターゲット層 / 強み・特徴 / 店舗展開 行: 各社 ※ 出典URLも列に入れてください 2. summary.docx — 1ページのサマリー 3社の違いがひと目で分かるように、簡潔にまとめてください。 最後に「どの企業がどんな顧客に向いているか」の所感を入れてください。 出力先は /outputs フォルダにしてください。
つまづきポイント:
Web検索が入るため、ファイルだけを読む課題②より少し時間がかかることがあります。サイドバーで「検索中...」「情報を整理中...」と進捗が動いているのを観察しましょう。最初のプロンプトで「何を比較したいか」を明確に伝えるのがコツです。
観察ポイント: Coworkがどんな検索クエリを使っているか、サイドバーの進捗表示を見てみましょう。「自分だったらどう検索するか」と比べてみると面白いです。

6課題②（7分）アンケート50件の整理

kadai2_survey_responses フォルダには、ある架空サービスの利用者アンケート50件の自由記述回答が入っています。全部読んで、頻出テーマごとにグルーピングし、テーマごとに代表的な声を引用しながら1枚のレポートにまとめてください。

期待される成果物:

  • テーマ別まとめレポート（.docx）

プロンプト例:

kadai2_survey_responses フォルダ内の50件のアンケート回答を全部読んで、 以下の構成でレポート（report.docx）を作ってください。 1. エグゼクティブサマリー（3行） 2. 抽出された主要テーマ（3〜4つ） 各テーマについて: - テーマ名と該当件数 - 代表的な声を2〜3件引用 - そのテーマから読み取れる示唆 3. 全体考察 出力先は /outputs フォルダにしてください。
つまづきポイント:
50件のファイルを読むのに少し時間がかかります。サイドバーで進捗バーが動いているのを確認しましょう。Coworkが「全件読み終わりました」と言うまで待つのがポイントです。

7後半（8分）自分の業務に置き換えてみよう

ここからは、自分の業務データを使って試します。以下の例を参考に、あなた自身の業務課題を選んでください。

例: フォルダの棚卸し 手元の「複数ファイルが入ったフォルダ」を1つ選んで、中身の棚卸しとサマリーを作らせる
例: Webリサーチ 業界の競合3社やトレンドなど、テーマを決めてWebで調べさせ、比較表やレポートを作らせる

もちろん、上の例にとらわれず他の作業を試しても構いません。普段時間がかかっている作業をCoworkに任せてみましょう。

作業が終わったら、ワークシートに以下の3点を書き留めてください。

  1. 何を任せたか
  2. どこまで自動でできたか
  3. どこで人間の判断が必要だったか
注意: 機密情報を含むフォルダは避けてください。まずは社外秘でない資料で試すのが安全です。
まとめ

8明日から使うためのヒント

  • 「フォルダを決めてから頼む」
    Coworkはプロジェクト単位で動きます。まず作業フォルダを指定してから指示を出しましょう。
  • 「成果物の形式を指定する」
    .xlsx / .docx / .pdf など、欲しいファイル形式を明示すると、期待通りのアウトプットが得られます。
  • 「最初に計画を一緒に詰める」
    いきなり作業を任せるより、まず「やりたいことの計画をCoworkと一緒に整理」してから作業に入ると、手戻りが減ります。「まず計画だけ出して」と頼むのがコツです。
  • 「途中で口を挟める」
    サイドバーから進捗を見て、いつでも軌道修正できます。走り出してからでも止められます。
  • 「大事なファイルの削除・上書きに注意」
    Coworkはファイルを作成するだけでなく、既存ファイルを上書きすることもあります。重要なファイルは作業フォルダの外に置くか、バックアップを取ってから作業しましょう。
